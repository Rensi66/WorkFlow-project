Workflow Task Tracker

Workflow — это лаконичный и эффективный сервис для управления задачами. Он помогает пользователям структурировать свой рабочий день, записывать текущие дела и контролировать статус их выполнения в реальном времени.
🎯 О проекте

Проект решает проблему хаоса в рабочих процессах. Это не просто «список дел», а бэкенд-система, которая позволяет хранить задачи, отслеживать их жизненный цикл (от создания до завершения) и гарантировать, что ни одна деталь не будет потеряна.
🛠 Технологический стек

Проект построен на современном стеке, который часто встречается в реальной коммерческой разработке:

    Python 3.10+: Основной язык разработки.

    PostgreSQL: Профессиональная реляционная база данных для надежного хранения ваших задач.

    Alembic: Инструмент для управления миграциями (изменениями) базы данных.

    Docker & Docker Compose: Позволяют запустить весь проект со всеми зависимостями одной командой, не засоряя вашу систему.

🚀 Как запустить проект у себя

Вам не нужно устанавливать Python или базу данных вручную. Достаточно иметь установленный Docker.
1. Клонирование
Bash

git clone https://github.com/Rensi66/WorkFlow-project.git

2. Запуск одной командой

Запустите сборку и старт всех сервисов (база + само приложение):
Bash

docker compose up -d --build

3. Работа с API

Как только контейнеры поднимутся:

    Сервис будет доступен по адресу: http://localhost:8000/workflow

    Интерактивная документация (Swagger): http://localhost:8000/docs — здесь можно потыкать эндпоинты, создать свою первую задачу и сменить ей статус прямо из браузера.

🏗 Что внутри (для разработчика)

    Контейнеризация: Приложение полностью изолировано.

    Авто-миграции: При старте Docker сам накатит все таблицы в базу данных.

    Clean Code: Логика разделена так, чтобы проект было легко расширять (например, добавить категории задач или дедлайны).
